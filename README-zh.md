# Full Stack Open

这是我在`University of Helsinki`的`全栈公开课`中所完成的练习。

该课程是介绍如何使用 JavaScript 开发现代Web应用程序。课程的重点是使用 ReactJS 构建单页面应用程序（SPA），并使用由 Node.js 构建 REST API。该课程还包含介绍 GraphQL 的部分，这是 REST API 的现代替代方案。课程还包括测试、配置和环境管理，以及使用 MongoDB 来存储应用的数据等等。

## Q&A

1. Q：为什么要记录时间线？

    A：因为课程的材料会随时间变化，这样做方便以后我自己看（如果我还会看的话），毕竟我学习这个课程期间偶尔会因为其他事情而不得不暂停。

2. Q：有什么遗憾吗？

    A：有的有的，就是我没有选择按 Exercise 提交。我最开始的时候就在考虑这个仓库的提交规范，想在按 Exercise 提交（即每个 Exercise 提交一次）和按 Part 提交（即每个 Part 提交一次）中选一个。当时觉得按 Exercise 提交会导致提交次数很多，然后我个人比较喜欢能简则简，所以就选了按 Part 提交。但是，现在想想，按 Exercise 提交的话，Git 就能帮我记录我做练习的过程，比如一个练习加了哪些代码、改了哪些代码都能清楚，还能更详细的知道每个 Part 的完成时间段而不仅仅是时间点。
